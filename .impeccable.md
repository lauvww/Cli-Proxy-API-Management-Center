## Design Context

### Users

This management center is primarily used by individual operators and small technical teams running their own CLI Proxy API deployments. They use it to configure the proxy, manage auth pools and credentials, inspect usage, and troubleshoot runtime issues without leaving the browser.

### Brand Personality

Warm, composed, capable.

The interface should feel like a refined operations console rather than a generic admin panel. It should align with the user's "日照金山" theme preference: early sunlight, mountain stone, warm gold highlights, steady confidence, and calm control instead of cold dashboard minimalism.

### Aesthetic Direction

Use a high-quality control-panel aesthetic adapted to a sunrise-over-mountains palette. Surfaces should feel layered and intentional, with restrained warm highlights, misted depth, and clear information density. Preserve the existing product patterns and IA, but make key panels feel more premium and less boilerplate.

### Design Principles

1. Keep operator workflows direct: surface the important controls and state first, remove explanatory filler when the same space can hold live data.
2. Use warm gold, stone, and haze accents sparingly for emphasis; avoid neon or generic dark-tech styling.
3. Prefer compact, high-signal UI with strong hierarchy over decorative cards and redundant helper banners.
4. New visual polish must remain practical: configuration, usage, and auth management should still feel fast and trustworthy.
